1. System architecture

  • Firebase Auth — Email/password + Google; session via FirebaseAuth; routing in AuthRouterActivity.
  • Cloud Firestore — Collections: users, listings, favorites; persistence enabled in StayFinderApplication.
  • FCM — StayFinderFirebaseMessagingService; token stored on users.fcmToken.
  • Fragments — Home, Map, Profile, Host, Bookings, Detail; auth: LoginFragment, RegisterFragment.
  • Jetpack Compose — Favorites tab: FavoritesComposeRoute in ui/compose/, hosted by FavoritesFragment.

2. Feature → code mapping

F1 Auth     → auth/AuthRouterActivity, auth/AuthActivity, auth/LoginFragment, auth/RegisterFragment, firebase/AuthRepository
F2 Firestore → firebase/FirestoreListingRepository, FirestoreFavoritesRepository, FirestoreUserRepository
Session      → AuthRouterActivity (launcher), MainActivity onStart guard
FCM          → messaging/StayFinderFirebaseMessagingService, messaging/AppNotificationCenter, StayFinderApplication (token sync)
Compose UI   → ui/compose/FavoritesComposeScreen.kt, Theme.kt, FavoritesFragment.kt
Maps upgrade → MapFragment.kt + Firestore listing coordinates
Search/filters → HomeFragment.kt + fragment_home (chips, price, rating)
Recommendations → features/RecommendationEngine.kt + HomeFragment horizontal list
Offline mirror → features/offline/OfflineFavoriteMirror.kt (SharedPreferences backup; Firestore SDK handles primary offline cache)

3. Self-researched features

A. Maps integration (listing markers)

Why: Show real Firestore listings on an embedded Google Map with custom markers.

Docs: Maps SDK for Android — Add a map, Markers

B. Search + filters + real-time Firestore UI

Why: Filter by text, max price, min rating, and category chips while the home feed syncs live from Firestore snapshots.

Docs: Listen for realtime updates, Queries (client-side filtering used to avoid index setup in class projects)

C. Offline favorites mirror (extra)

Why: Demonstrates a local mirror of favorite ids alongside Firestore offline persistence.

Docs: Firestore offline persistence

D. Recommendation heuristic

Why: Lightweight “for you” strip based on favorited property types and tags.

Docs: N/A (custom logic); related: Android recommendation overview

4. Flow (auth → main → logout)

Launcher: AuthRouterActivity
  ├─ user != null → MainActivity (bottom nav, Firestore listeners)
  └─ user == null → AuthActivity → LoginFragment / RegisterFragment
        → success → MainActivity

Logout: ProfileFragment → GoogleSignIn.signOut + AuthRepository.signOut → AuthRouterActivity

5. Firestore shape (reference)

users/{uid}     — email, displayName, fcmToken, createdAt
listings/{id}   — title, description, pricePerNight, priceDisplay, location, lat/lng, ownerUid, tags[], rating, ...
favorites/{userId_listingId} — userId, listingId, note, denormalized title/price/location/imageUrl/rating, updatedAt
